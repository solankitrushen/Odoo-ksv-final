# PRODUCT.md — Renton Admin (master-admin)

## Register

**product** — design serves the ops task.

## Users

Non-tech rental operators (owner, counter staff). Day-to-day workflow: clear today’s work, open records in one click, run plain-language actions.

## Product purpose

Rental Portal admin for managing customers, products, rentals, today’s pickups/returns/deliveries, and payments against the rental admin API.

## Brand personality

Utilitarian, calm, dense-enough. Plain words over ERP jargon.

## Anti-references

InstantCafe / Clerk leftover copy · purple SaaS glow · VendorBridge naming · multi-step wizards for simple list→detail · “View” link as only row affordance.

## Strategic principles

1. Whole-row click → detail whenever detail exists.
2. Minimize clicks (list → detail = 1).
3. One primary CTA per list page.
4. Lifecycle verbs on rental detail in human language.
5. Spec-first FE wiring under `docs/specs/admin-ui-*`.
